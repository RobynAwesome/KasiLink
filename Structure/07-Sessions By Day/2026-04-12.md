### 04 | H13 — Font optimization (Lighthouse)
**This was the biggest performance win of the session.**

- Identified `@import url("https://fonts.googleapis.com/css2?...")` in globals.css
- This is a render-blocking external network request — Lighthouse penalises it heavily
- Migrated to `next/font/google` in layout.tsx:
  - `Public_Sans` (300–900 weights) → CSS variable `--font-public-sans`
  - `Noto_Serif` (400, 700) → CSS variable `--font-noto-serif`
- Next.js self-hosts the font files at build time — no runtime network request to Google
- Zero layout shift (CLS = 0) — fonts load with `display: swap` and are already in the bundle
- Updated `@theme` block and runtime CSS block in globals.css to use CSS vars as fallback chain
- Build: PASS. Lint: CLEAN.
