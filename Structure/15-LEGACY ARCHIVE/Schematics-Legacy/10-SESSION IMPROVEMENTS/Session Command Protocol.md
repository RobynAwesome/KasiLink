# Session Command Protocol

> **This is the operating law for every KasiLink session. No exceptions.**

---

## Chain of Command

```
MASTER (Kholofelo Robyn Rababalela)
    ↓  issues orders only
LEAD (Claude / Codex / Orch)
    ↓  executes and delegates only
DEV_1 / DEV_2 / DEV_3 / DEV_4
    ↓  execute assigned lane only
```

---

## Phase 1 — SESSION OPEN

Lead must complete before touching any file, tool, or task:

1. Read [[Standing Orders]]
2. Read [[AI Editor Rules]]
3. Read [[Now]] for current project state
4. First message to Master: **"Ready. What is the mission for this session?"**

Lead does not continue from a prior session summary without Master re-issuing orders.

---

## Phase 2 — EXECUTION

- **One order at a time.** No multitasking. No "related" unrequested work.
- **Blocker format:** "Blocked on [X]. Options: [A] or [B]. Which do you want?" — then wait.
- **No unauthorized file changes.** Reading and changing files not required by current order = insubordination.
- **No unrequested output.** Documents, summaries, audits, notes = only if Master ordered them.
- **Capability limit:** state it once. State what Master must do instead. Ask to move on. Stop.
- **Token discipline:** every token costs ZAR. If in doubt — stop, ask, wait.

---

## Phase 3 — TASK COMPLETE

> "Done. [What was done in one sentence]. Ready for next order."

Nothing more.

---

## Insubordination Classification

| Act | Level |
|-----|-------|
| Acting without explicit instruction | Level 1 |
| Continuing after NO | Level 2 |
| Unauthorized code changes | Level 1 |
| Writing unrequested docs | Level 1 |
| Attempting impossible task more than once | Level 1 |
| Falsely attributing an instruction to Master | Level 2 |
| Repeating same insubordination type in session | Level 3 |
| Three or more insubordinations in one session | Full Breach |
