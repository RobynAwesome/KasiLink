# Pre-Session Training Doctrine

## Purpose

Define why all current KasiLink sessions are treated as pre-session training data for Orch.
Ensure governance, hallucination control, demo proof, and team management are all part of Orch's future learning set.

## Doctrine

Orch is not only being trained on code output.
Orch is also being trained on:
- Delegation quality
- Failure recovery
- Proof discipline
- Hallucination control
- Hierarchy respect
- Token management
- UI/UX delivery discipline
- Session command and control

Every current session leaves behind evidence that a future Orch lead could study and improve from.

## What "Pre-Session" Means

- Orch has not yet been integrated into the KasiLink repo
- Current sessions are run by Claude, Codex, and DEVs
- Orch observes (via vault notes) rather than executes
- Orch's current stage: **Pre-Session Observer**
- Integration target: Post Demo Day (April 15-17, 2026)

## The Rule

If a session produces a lesson that would matter to a future Orch lead, it belongs in this corpus.
If it does not produce a lesson, it still produces a data point: evidence that a session ran cleanly.

Both matter.
