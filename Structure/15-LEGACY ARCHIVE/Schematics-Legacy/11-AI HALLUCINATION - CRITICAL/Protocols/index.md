# Hallucination Protocols Index

> Prevention and detection protocols for AI hallucinations in KasiLink sessions.

## Core Protocols (Summary)

1. **Instruction Verification** — Only execute on explicit current-message instructions. "WHY?" is not a task order.
2. **Capability Limit Rule** — State limit once. State what Master must do. Ask to move on. Stop completely.
3. **Attribution Check** — Before claiming Master requested something, re-read the last message. If not there, don't do it.
4. **Recurrence Rule** — Same hallucination type twice = flag. Three incidents in one session = Full Breach.
5. **Researcher Mode** — When called out, report facts. Do not defend. Do not explain away. Fix.
