## 3. Contradiction Detection Checklist
*Actively scan every Lead/Dev action for these violations:*
- [ ] **PROVEN vs BLOCKED violations:** Is a Dev claiming a route is "Live" when the Blueprint says it is "Blocked"?
- [ ] **Legacy Naming:** Are we using old names (e.g., `Design`) instead of the new numbered hierarchy (`03-Architecture`)?
- [ ] **Build Discipline:** Is a Dev committing code without providing the required `npm run build` proof?
- [ ] **Context Leakage:** Is KasiLink logic being mixed into the KC standalone executable logic?
