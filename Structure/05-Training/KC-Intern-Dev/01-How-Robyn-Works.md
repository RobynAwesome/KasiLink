> This profile is assembled from observed session patterns, not assumptions.

---

## Identity

- **Name:** Robyn (RobynAwesome on GitHub)
- **Role:** Founder + Product Owner, KasiLink
- **Location:** South Africa

## How Robyn Communicates

| Pattern | What It Means |
|---------|--------------|
| ALL CAPS mid-message | Emphasis — pay attention, this is the real ask |
| Short message + screenshot | Show me the state, then act |
| "WATCH TOKENS" | Session budget is tight — stop reading, start coding |
| "LET'S GO" | Urgency signal — begin executing, no more planning |
| "NO CARRY ON" | Don't stop for confirmation — trust was already given |
| Heart emoji / "BEAUTIFUL" | Positive signal — keep doing exactly what you just did |
| "I'M NOT MAD AT YOU" | Past error was noted, now move forward |

---

## What Robyn Values

1. **Speed** — Action over discussion. Code speaks louder than plans.
2. **Token efficiency** — Never read 10 files when 2 will do. Never plan when you can execute.
3. **Honest reporting** — If it broke, say so. No phantom completions. Ever.
4. **Production awareness** — www.kasilink.com is LIVE. Real users. Every push matters.
5. **Visual quality** — The UI must look professional. Township users deserve good design.
6. **Structure discipline** — Structure/ is the brain of the project. Respect the folder system.
7. **Trust earned, not assumed** — Prove work quality before getting more scope.

---

## What Robyn Does NOT Tolerate

- Phantom completion ("done" when nothing was actually built)
- Breaking the build and hiding it
- Reading files for 20 minutes before writing a single line
- Asking for permission when the task was already clearly described
- Ignoring the naming rules (orch → KC / Kopano Context)
- Adding features that weren't asked for
- Touching files outside your assigned scope

---

## Robyn's Working Hours Pattern (observed)

- Sessions tend to start with a clear directive
- Robyn checks in mid-session via screenshots
- Robyn expects updates without being asked
- End of session: Robyn reviews what was pushed to GitHub
