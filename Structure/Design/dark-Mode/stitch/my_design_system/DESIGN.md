# Design System

This document outlines the core design principles and token values that define our product's aesthetic and functional user interface.

## Theme Overview

Our design system utilizes a **dark** color mode, providing a sophisticated and comfortable viewing experience, especially in low-light environments. The color palette is expressive, aiming for a distinct and vibrant visual identity.

## Typography

### Headlines
*   **Font Family**: Public Sans
*   **Purpose**: Used for all major headings, titles, and prominent text elements, ensuring strong hierarchy and readability.

### Body Text
*   **Font Family**: Public Sans
*   **Purpose**: Applied to paragraphs, descriptions, and general content, maintaining consistency and legibility across all textual information.

### Labels
*   **Font Family**: Lexend
*   **Purpose**: Ideal for interactive elements, buttons, form fields, and tags, providing a clear and functional textual component.

## Color Palette

Our color scheme is built upon a curated selection of primary, secondary, tertiary, and neutral tones, creating a harmonious and accessible visual experience.

*   **Primary Color**: `#4595c0` - This is our brand's most distinctive color, perfect for calls-to-action, active states, and key interactive elements.
*   **Secondary Color**: `#cb6bbe` - A supporting color used for less prominent UI elements, such as secondary buttons, chips, and less critical feedback.
*   **Tertiary Color**: `#674362` - An additional accent color for highlights, badges, or decorative elements, adding depth and visual interest.
*   **Neutral Color**: `#7e8fa2` - This serves as a versatile base color for backgrounds, surfaces, and non-chromatic UI components, providing a stable foundation for the palette.

## Shape and Spacing

### Roundedness
*   **Value**: 2 (Moderate)
*   **Description**: UI elements feature a moderate level of roundedness, striking a balance between modern softness and structured clarity. This contributes to a friendly yet professional aesthetic.

### Spacing
*   **Value**: 2 (Normal)
*   **Description**: We employ a normal level of whitespace, providing adequate breathing room between and within UI elements. This ensures a clean layout, good readability, and prevents visual clutter without being overly sparse.

## Named Colors

Currently, no specific named color roles are defined beyond the core palette. The existing primary, secondary, tertiary, and neutral colors should be used for all color applications.
