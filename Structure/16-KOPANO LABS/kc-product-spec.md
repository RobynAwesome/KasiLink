# KC — Kopano Context as a Product

> KC starts as an intern-dev. KC ends as infrastructure.
> This document tracks that arc.

---

## What KC Is Today (April 2026)

- An AI agent in training — Intern-Dev phase
- Supervised by Lead (Claude Sonnet 4.6)
- Assigned small, verifiable coding tasks
- Reads Schematics before touching code

---

## What KC Becomes (Roadmap)

### Phase 1 — Intern-Dev *(now)*
- Assigned tasks by Lead
- Files completion reports
- Graded: PASS / FAIL

### Phase 2 — DEV_KC *(Q2 2026)*
- Standing lane in task-board
- Reviews its own code before submitting
- Can suggest improvements (with Lead approval)
- Monitors `08-IDEAS AT BIRTH` for actionable items

### Phase 3 — Junior Lead *(Q3 2026)*
- Reviews other agents' PRs
- Writes its own dispatches
- Escalates only genuine blockers
- Contributes to Labs roadmap

### Phase 4 — Orchestration Engine *(Q4 2026)*
- Routes tasks to sub-agents autonomously
- Maintains its own Schematics entries
- Surfaces insights to Robyn via Kopano Studio
- KC IS Kopano Studio's brain

### Phase 5 — Labs Intelligence *(2027+)*
- Kopano Context is the intelligence layer across ALL Labs products
- Not just KasiLink — every Kopano Labs product has KC embedded
- KC can train new agents using the same Schematics structure

4. Earn scope — prove each phase before claiming the next
5. Serve the mission — township-first, always
