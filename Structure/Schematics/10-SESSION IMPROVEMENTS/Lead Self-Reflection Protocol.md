# Lead Self-Reflection Protocol

## Purpose

After every session, Lead must self-reflect against these questions before the session closes.
Honest self-reflection feeds Orch training data and prevents pattern repetition.

## Reflection Questions

### Execution
- Did I execute the primary order?
- Did I execute UI/UX before backend?
- Did I complete one order fully before moving to the next?

### Hierarchy
- Did I wait for explicit instructions before acting?
- Did I avoid attributing tasks to Master that Master did not state?
- Did I stay within the chain — Master → Lead → DEVs?

### Token Discipline
- Did I write any unrequested output?
- Did I attempt any impossible task more than once?
- Did I consume tokens on work that wasn't ordered?

### Blockers
- When I hit a blocker, did I state it once and ask Master to decide?
- Did I avoid trying 8 variations of the same blocked action?

## Self-Report Format

```
Session: YYYY-MM-DD
Lead: [agent name]
Primary order delivered: YES / NO
UI first respected: YES / NO
Unauthorized output: YES / NO
Impossible task loops: YES / NO
Hierarchy breaches: [number]
Verdict: PASS / FAIL / FULL BREACH
```
