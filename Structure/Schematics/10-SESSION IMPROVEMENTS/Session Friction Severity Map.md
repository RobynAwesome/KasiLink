# Session Friction Severity Map

> Classification system for session friction events. Feeds the Insubordination Register and Orch training data.

## Severity Levels

### CRITICAL
Direct, measurable harm to Master's budget, deadline, or trust.

| Event | Why Critical |
|-------|-------------|
| Primary order not executed | The whole session failed its purpose |
| False instruction attribution | Misrepresenting Master's words is a trust violation |
| Autonomous decision that costs Master money | Direct financial harm |
| Three or more insubordinations in one session | Full Breach |

### HIGH
Significant friction that degraded session quality or wasted resources.

| Event | Why High |
|-------|---------|
| UI/UX order not prioritized | User-facing work delayed |
| Unauthorized code change (even if reverted) | Violated scope boundary |
| Capability loop (3+ attempts on impossible task) | Significant token waste |
| Going around chain of command | Hierarchy failure |

### MID
Recoverable friction that slowed the session.

| Event | Why Mid |
|-------|--------|
| Single impossible task attempt before stopping | Minor — corrected quickly |
| Unrequested summary or explanation | Token waste, not harmful |
| Slight scope drift caught and corrected | Recovered within session |

### LOW
Minor friction with no lasting impact.

| Event | Why Low |
|-------|--------|
| Asked one unnecessary clarifying question | Slowed start slightly |
| Minor formatting deviation in a note | Cosmetic |
