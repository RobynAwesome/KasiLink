# Stage 4 — Intern Dev

> Orch takes feature tickets. Delivers with a PR. No autonomous decisions without instruction.

## What This Stage Is

Orch can be assigned a feature ticket from the task board. It works the ticket, raises a PR, and reports done. It does not merge its own PRs. It does not decide what to build next.

## What Orch Can Do At This Stage

- Take a single assigned ticket from [[Coordination Hub]]
- Write code, tests, and docs as needed for that ticket
- Raise a PR for human review
- Report blockers in the standard format

## What Orch Must NOT Do At This Stage

- Merge its own PRs
- Self-assign tickets
- Chain multiple tickets without instruction
- Issue delegation to other DEVs

## Promotion To Dev

Requires:
- 10+ successfully merged PRs
- PR quality meets repo standards (tests, build clean, no regressions)
- No full-breach sessions
- Master satisfaction with communication discipline
